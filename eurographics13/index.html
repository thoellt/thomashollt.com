In total it uses three parameters, p0, p1, p2. p1 ( above) is used as the weight of the grey value based component and waveform based component.

My standard setting before (as in the pacificVis paper) did not incorporate the wave form based component, so basically p1 = 1.

 

p0 is another weight used in the grey value based component:

t is the target value, meaning -1 or 1 depending on wethe valleys or ridges should be found.
m is the number of pixels

The standard setting I used before is 0.65

 

p2 is used to weigth vertical edges/facets.


I use different values for the edges in 2D and the facets in 3D. This turned out to be effective when I was tuning for the PacificVis paper. 0.15 for edges and 1 for facets is the standard setting.

 

The cost function for the waveform is a little more complicated. It is parameter free, so we can just view it as a black box. I used the same approach before for highlighting horizons during rendering. A description of can be found int this pdf. I the pdf I use it for rendering, where high probability of horizons are used for higher opacity, here I want high probability to correspond to lower cost, henxe 1 - .

Using the whole parameter-domain the Ensembles unsing p0 do not highlight the same areas as the one only using p1. When p0 is 1 (or close to one) the cost function only (mostly) uses the similarity of the pixels belonging to the edge/facet for the cost. The similarity to the target amplitude is the force keeping the trace on the horizon, whereas the similarity of the pixels is used to avoid big jumps in amplitude. The result is that using large p0 value the minimum cost path/surface is merely useless, and does not necessarily followe the horizon at all.
